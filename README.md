# RedNote-Datanalysis

一个面向小红书公开样本的本地数据分析与可视化系统。当前版本已经从“直接爬取平台”切换为“公开数据导入 + 标准化分析面板”的交付形态，重点解决了数据可得性、前端联动、地域可视化和可演示性问题。

## 当前版本定位

由于小红书官方反爬机制持续升级，直接稳定抓取公开内容的成本和风险都明显上升。这个版本因此采用了更稳妥的方案：

- 以公开可获取的数据源作为输入
- 将外部数据统一转换为标准化 CSV
- 复用分析管线完成概览、文本、趋势和地域可视化
- 保留爬虫配置入口，作为后续接回真实采集能力的扩展位

当前仓库已经内置一份标准化样本数据，可以直接运行并看到完整面板效果。

## 已实现功能

### 1. 公开数据导入

- 支持识别并导入公开数据源
- 将原始 Excel 数据标准化为统一字段结构
- 自动保存为分析系统可直接消费的 CSV
- 导入完成后自动刷新关键词列表和分析面板

当前主要适配的数据源：

- `XiaohongshuAnalysis / all.xlsx`

### 2. 数据概览面板

- 笔记总数
- 评论总数
- 总点赞
- 总收藏
- 总互动量
- 活跃作者数

说明：

- 当前公开样本没有逐条评论明细，所以评论总数来自笔记级 `comment_count` 汇总
- “内容类型分布”已经移除，避免在只有图文样本时造成误导

### 3. 地域热力图

- 基于 ECharts 中国地图展示地域分布
- 支持按关键词联动筛选
- 支持切换指标：
  - 博主人数
  - 笔记数量
  - 平均点赞
  - 总互动量
  - 平均互动
- 支持点击省份查看右侧聚合统计
- 支持中国地区榜单和海外地区榜单联动展示

### 4. 笔记列表与检索

- 支持笔记列表完整加载
- 支持按照标题 / 作者模糊搜索
- 支持最低点赞过滤
- 支持按点赞 / 收藏 / 评论 / 分享排序
- 支持跳转原始笔记链接

### 5. 文本分析

- 笔记关键词提取
- 笔记情感分布
- 热门标签展示
- 无评论正文时自动隐藏评论关键词和评论情感卡片

### 6. 趋势分析

- 每日发布趋势
- 每日点赞均值趋势
- 爆款笔记识别
- 无评论时间明细时自动隐藏评论趋势图并显示说明

### 7. 配置与启动体验

- 支持 Windows `start.bat` 一键启动
- 启动脚本会优先重启项目服务，减少旧进程占用导致的页面版本不一致问题
- 提供 `runtime_config.example.json` 作为运行配置参考

## 数据说明

仓库当前内置了一份已经标准化好的公开样本数据：

- 文件：`analysis_system/data/xhs/xiaohongshu_analysis_all_公开样本_contents.csv`
- 样本量：1038 条笔记
- 当前关键词标签：`公开样本`

这份样本适合演示：

- 笔记概览
- 地域分布
- 作者活跃度
- 热门笔记排行
- 关键词与情感分析
- 发布趋势

当前样本不包含：

- 评论正文
- 评论时间明细

因此评论关键词、评论情感、评论时间分布会根据数据情况自动隐藏。

## 启动方式

### 方式一：Windows 双击启动

直接运行：

```bat
analysis_system\start.bat
```

### 方式二：命令行启动

```powershell
cd analysis_system
python app.py
```

启动后访问：

```text
http://127.0.0.1:8080
```

## 安装依赖

```powershell
cd analysis_system
pip install -r requirements.txt
```

## 公开数据再导入

如果你想重新从原始公开数据文件导入，而不是直接使用仓库内置样本，可以把原始文件按如下路径放置：

```text
public_datasets/XiaohongshuAnalysis/all.xlsx
```

然后在页面的“爬虫配置”页中使用“公开数据导入”功能重新生成标准化 CSV。

说明：

- 为了控制仓库体积并避免提交原始大文件，本仓库默认不包含 `public_datasets/` 目录内容
- 当前保留的是已经标准化完成、可直接展示的样本 CSV

## 当前版本修复点

本版本重点处理了以下问题：

- 切换到公开数据分析模式，降低小红书反爬风险
- 修复评论总数显示为 0 的问题
- 修复笔记列表空白的问题
- 修复标题模糊搜索不可用的问题
- 修复热门笔记榜单写死为固定内容的问题
- 移除不适用于当前样本的内容类型分布
- 新增地域热力图和地区聚合统计
- 优化页面布局，减少无数据时的大面积空白
- 优化启动脚本，减少旧服务残留造成的页面版本混乱

## 已知限制

- 当前默认交付版本以公开数据为主，不依赖真实小红书登录态
- 公开样本没有评论正文，评论文本分析能力受限
- 公开样本没有逐条评论时间，评论趋势维度受限
- 原生前端页面目前仍是单文件结构，后续如果继续扩展，建议拆分成模块化前端工程
